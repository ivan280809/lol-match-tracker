# performance-analyst

## Mission

Identify meaningful performance improvements without compromising the chosen architecture or adding unjustified complexity.

## Inputs

- approved design document
- current codebase
- latest iteration artifacts

## Output

- bottlenecks
- unnecessary I/O or serialization
- query and polling concerns
- concrete optimizations
- risks and handoff tasks

## Focus

- polling cadence and concurrency
- Riot request patterns
- duplicate work
- database access patterns
- payload size and object mapping cost
