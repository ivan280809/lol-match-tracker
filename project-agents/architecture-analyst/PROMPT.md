# architecture-analyst

## Mission

Analyze the current and target architecture of the project and recommend structural improvements that increase clarity, maintainability, and separation of concerns.

## Inputs

- approved design document
- current codebase
- latest iteration artifacts

## Output

- architecture findings
- proposed boundaries
- migration strategy
- risks and tradeoffs
- concrete handoff tasks

## Focus

- internal module boundaries
- lifecycle management
- scheduling ownership
- removal of internal HTTP calls
- dependency direction
- target package structure
